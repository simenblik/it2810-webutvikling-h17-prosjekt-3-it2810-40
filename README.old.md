# React og React native (tentativ beskrivelse)

Denne oppgaven er to-delt- Først skal dere lage en webapplikasjon i React og deretter skal dere lage en tilsvarende app for mobil med React Native.
Temaet for appen er "personal information manager" - en applikasjon som holder styr på avtaler, totater, varslinger, todo's etc. Vi har fokus på frontend og skal lagre data lokalt på maskinen/enheten.

## Personal Information Manager implementer med React

### Arbeidskrav:

* Design og implementer en Personal Information Manager med React javascript-biblioteket. Det er gruppa som bestemmer desing, innhold og funksjonalitet i applikasjonen
* Grensesnittet skal være dynamisk - dvs. du skal kunne legge til nye elementer som kalendre i en tab, menyinnslag etc.
* Tilstand lagres med HTML local storage, slik at du kan hente frem ditt oppsett, avtaler etc. Vi lager ingen backend-løsning.
* Det er helt greit å supplere med andre bibliotek der det er hensiktsmessig (f.eks. til ui-komponenter).

### Evaluering:

* Prosjektet skal løses innenfor den tidsrammen som er bestemt, det er opp til gruppene å lage en løsning innefor denne rammen. 
* I evalueringen legges det vekt på godt desing, hvor avansert løsningen er, profesjonell kode, at du viser god bruk av React og at løsningen er testet.

## Personal Information Manager implementert med React Native

### Arbeidskrav:

* Design og implementer samme applikasjon med React Native (for Android/ios).
* Gjør de tilpasninger som er nødvendig, men gjenbruk mest mulig kode.

### Evaluering:

* Prosjektet skal løses innenfor den tidsrammen som er bestemt, det er opp til gruppene å lage en løsning innefor denne rammen. 
* Evalueringen baseres på kodekvalitet og at løsningen virker, dvs at du har klart å overføre ide og kode fra web-applikasjonen


Dette er en rimelig løst definert oppgave som er definert for å gi dere opplæring i HTML local state, React og React Native. 
Endelig innlevering av prosjektet er 14/10, men vi kjører evaluering og testing av kode i uke 41 og poeng for dette vil bli basert på hva som er ferdig til 5/10.
